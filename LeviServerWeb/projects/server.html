My Home Server

My home server is a versatile machine capable of hosting various services.

Hardware Specifications

  • Motherboard: ASUS P7F-M uATX (featuring 6 SATA ports and multiple PCIe expansion slots)
  • Processor: Intel Xeon X3450 (4 physical cores, 8 virtual cores with VT-x support)
  • Memory & Storage: 16GB DDR3 RAM, 64GB SSD (for the OS), plus 500GB and 160GB HDDs
  • Graphics: ATI Radeon X550 Advantage GPU (connected via VGA)
  • Case & Drives: ATX Maxdata case with HDD/SSD slots, DVD/CD drive (via IDE-to-USB adapter), and a floppy drive (via FDD-to-USB adapter) allowing me to use floppies and disks
  • Networking & Power: Small USB Wi-Fi adapter for wireless connectivity, powered by a Be quiet! 430W PSU with standard cooling

Software & Services

The server runs Ubuntu because Linux is lightweight, stable, and ideal for older hardware while still capable of running all my services:

  • Website Hosting: Hosts this very website, featuring a script that pulls the latest code from my GitHub repository every 30 seconds, so changes go live without manual restarts.
  • NAS Storage: Acts as a local NAS using CasaOS, letting me store and access files from any device at home.
  • OpenWebRX: A publicly accessible software-defined radio (SDR) receiver that allows listening to radio signals via the connected SDR device.
  • Public Telnet Server: A terminal-accessible server responding to specific commands (built as a fun project); accessible via the demo page or by using a terminal with details found on the demo page.
  • Remote Management: Fully accessible remotely via SSH and VNC.

Network Architecture & Public Access

Because my internet plan uses DS-Lite and lacks a public IPv4 address, I implemented a workaround to make my services public:

  • Tunneling: A private encrypted Tailscale tunnel connects my home server to an AWS EC2 instance.
  • Reverse Proxy: Domain and subdomain A records point to the EC2's public IPv4. An Nginx instance on the EC2 on port 80 proxies requests (like ishir. or sdr.) through the Tailscale tunnel to the correct port on my home server.
  • Port Forwarding: socat runs on the EC2 instance to forward direct connections like SSH, Telnet and Websockets (e.g., my telnet service) straight to my home server's Tailscale IP.
